Git Essentials

Git -Version control software has “Distributed“ repository model.

The repository model available are:

  1. Distributed - every user has a local copy of the repository in addition to the central repo on the server side. Example: Git
  2. Client-Server-the repository is located one place. Example: SVN (Apache SVN)
  3. Local – only local copy. Example: Source Code Control System
  4. Distributed and Client server. Example: Azure DevOps

SSH SSL
Securely and remotely connecting to another machine to issue commands
Network tunnels 		securely transmitting data between two parties.
SSL certificate
Requires the client to authenticate with a username/password or cryptographic key Only requires authentication on the server side (the client isn’t required to authenticate)

Configurations are:

  1. System: git config –system
  2. User: git config –global
  3. Project: git config

Git init creates .git folder with all configuration required files.

Two repeated processes: AddCommit

The latest commit message store in the COMMIT_EDITMSG file.

Git logs are stores in the .git/logs/refs/heads/master file. As well as, in the .git/logs/HEAD
